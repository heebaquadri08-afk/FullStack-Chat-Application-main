ABSTRACT

The FullStack Chat Application is a real-time messaging platform designed to provide seamless and instantaneous communication between users. Built using the MERN stack (MongoDB, Express.js, React, and Node.js), the application leverages Socket.io to achieve bi-directional, low-latency communication. Key features include secure user authentication using JSON Web Tokens (JWT) and Bcrypt, a responsive user interface styled with Tailwind CSS, and real-time status updates such as online/offline tracking. The system also integrates Cloudinary for efficient profile picture and image message management. This report details the design, implementation, and various modules of the application, fulfilling the requirements for a modern full-stack web project.

CHAPTER-1

1.1 INTRODUCTION
The rapid evolution of web technologies has transformed the way people communicate. Modern chat applications require high responsiveness and real-time capabilities to maintain user engagement. This project, titled FullStack Chat Application, is an end-to-end solution that addresses the need for secure and instant messaging. By utilizing the power of WebSockets and the flexibility of the MERN stack, the application offers a robust environment for personal communication.

1.2 OBJECTIVES
The primary objectives of this project are:
To develop a real-time messaging system using WebSockets.
To implement secure user authentication and authorization.
To provide a responsive and user-friendly interface across devices.
To allow users to manage their profiles and customize application themes.
To ensure data persistence using a NoSQL database.

The project interface is a web-based application accessible via modern browsers. 1.4 BACKGROUND
Traditional messaging systems often relied on HTTP polling, which is inefficient and leads to significant latency. With the advent of HTML5 and WebSockets, real-time communication has become more accessible. This project builds upon these technologies to create a modern chat experience that handles concurrent connections efficiently.

Users often face issues with slow message delivery and lack of data security in basic chat implementations. Furthermore, many existing solutions are not mobile-friendly or lack personalization features. There is a need for a lightweight, secure, and real-time chat application that provides a consistent experience across all platforms.

1.6 AIMS AND OBJECTIVES
The aim of this project is to bridge the gap between simple chat scripts and complex enterprise messaging systems. The objectives involve creating a scalable backend using Node.js, a secure database schema in MongoDB, and a dynamic frontend in React that reflects state changes in real-time.

CHAPTER-4

4.1 PROBLEM DESCRIPTION
The project addresses the challenge of building a synchronized messaging environment where multiple users can interact without page refreshes. It involves managing socket connections, mapping user IDs to socket IDs, and ensuring that messages are correctly routed and stored.

4.2 MODULE 1: AUTHENTICATION AND AUTHORIZATION
This module handles user registration and login. It uses Bcrypt for password hashing to ensure security. Upon successful login, a JWT is generated and stored in a cookie. Middleware is implemented to protect private routes by verifying this token.

4.3 MODULE 2: REAL-TIME MESSAGING
This is the core module using Socket.io. When a user sends a message, it is saved to the database and simultaneously emitted to the receiver's socket ID if they are online. The frontend updates the message list instantly upon receiving the event.

CHAPTER-6

6.1 PROBLEM DISCUSSION
The main challenge during implementation was the synchronization of the message state when a user is chatting with someone. Ensuring that the socket is connected once the user is authenticated and disconnected on logout was crucial to maintain accurate "online" status lists.

6.2 IMPLEMENTATION OF PROBLEM
Implementation started with the database schema design. The messaging logic was built using an event-based approach:
1. Sender sends message via POST API.
2. Controller saves message to MongoDB.
3. Controller retrieves Receiver's Socket ID.
4. io.to(receiverSocketId).emit('newMessage', message) sends data.

9.1 CODING

Authentication Logic (auth.controller.js):
export const signup = async (req, res) => {
  const { fullName, email, password } = req.body;
  try {
    const salt = await bcrypt.genSalt(10);
    const hashedPassword = await bcrypt.hash(password, salt);
    const newUser = new User({ fullName, email, password: hashedPassword });
    if (newUser) {
      generateToken(newUser._id, res);
      await newUser.save();
      res.status(201).json(newUser);
    }
  } catch (error) {
    res.status(500).json({ message: "Internal Server Error" });
  }
};

Messaging Logic (message.controller.js):
export const sendMessage = async (req, res) => {
  try {
    const { text, image } = req.body;
    const { id: receiverId } = req.params;
    const senderId = req.user._id;
    const newMessage = new Message({ senderId, receiverId, text, image });
    await newMessage.save();
    const receiverSocketId = getReceiverSocketId(receiverId);
    if(receiverSocketId){
      io.to(receiverSocketId).emit("newMessage", newMessage);
    }
    res.status(201).json(newMessage);
  } catch (error) {
    res.status(500).json({ error: "Internal server error" });
  }
};

Socket Server Implementation (socket.js):
const io = new Server(server, { cors: { origin: [process.env.FRONTEND_URL] } });
io.on("connection", (socket) => {
    const userId = socket.handshake.query.userId;
    if(userId) userSocketMap[userId] = socket.id;
    io.emit("getOnlineUsers", Object.keys(userSocketMap));
    socket.on("disconnect", () => {
        delete userSocketMap[userId];
        io.emit("getOnlineUsers", Object.keys(userSocketMap));
    });
});
